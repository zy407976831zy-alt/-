# 实拍婚纱照 AI 商业精修工作流

## 核心目标

把摄影师实拍原片按参考图风格生成新场景，并达到可给客户看的商业婚纱精修标准。

硬性标准：

- 真人客户主体必须锁定，不能把客户重新生成成另一个人。
- 客户面部、发型主体、表情特征、身体姿态优先保留原片。
- AI 主要负责背景、花艺、场景氛围、局部婚纱延展和光影辅助。
- 最终必须通过原脸校准、真实光影融合、毛孔质感精修。
- 拒绝 AI 感、拒绝贴图感、拒绝一眼假。

## 项目目录

- `inputs/originals/`：摄影师实拍原片
- `inputs/references/`：参考风格图
- `outputs/01_backgrounds/`：单独生成或准备好的背景
- `outputs/02_composited/`：人物与背景合成初稿
- `outputs/03_face_locked/`：原脸回贴和身份校准后图
- `outputs/04_retouch_final/`：最终商业精修图
- `outputs/rejected/`：废片或不合格图
- `workflow/batch_jobs.csv`：批量任务表
- `templates/identity_preserve_prompt.md`：主体锁定和背景融合提示词
- `qa/quality_checklist.md`：质检清单
- `scripts/build_prompts.py`：批量生成提示词

## 正确生产链路

### 0. 两阶段风格迁移

当前优先采用用户在 GPT 中跑通的两阶段方法：

1. 先用参考图生成“风格提示词/风格包”。
2. 再只用客户原片和风格提示词生成最终图。

不要默认把参考图和客户图一起交给模型自由混合，否则容易污染客户脸、发型和身体特征。

详细说明见：

`workflow/two_step_gpt_method.md`

### 1. 原片入库

把客户实拍原片放入 `inputs/originals/`。

文件名建议：

`客户名_套系_序号.jpg`

### 2. 参考图入库

把参考风格图放入 `inputs/references/`。

参考图只允许提供：

- 场景方向
- 光线方向
- 色调
- 花艺/道具
- 构图氛围
- 发型参考，前提是任务表明确允许
- 服装参考，前提是任务表明确允许

参考图不允许提供：

- 客户身份
- 客户脸
- 客户身体比例

### 参考发型/服装开关

任务表里有两个控制项：

- `reference_hairstyle_option`
- `reference_wardrobe_option`
- `pose_expression_option`

可选值：

- `不参考`：不参考，保留原片。
- `参考风格`：只参考大方向，例如发型氛围、头饰、婚纱材质、袖型、头纱感觉。
- `匹配参考`：尽量贴近参考图，但仍然必须保留客户脸、头型、身体比例和真实姿态。

默认建议：

- 发型：`不参考`
- 服装：`参考风格`
- 动作表情：`动作保持`

使用红线：

- 参考发型不能改变客户脸型、发际线和年龄感。
- 参考服装不能破坏客户身体比例和姿态。
- 任何时候都不能复制参考图模特的脸和身份。

动作表情可选值：

- `动作保持`：保持原片动作、头部角度、眼神方向和表情，只换适配场景。
- `小幅调整`：小幅调整动作和表情，让人物更自然融入新场景。
- `重新设计`：允许重新设计动作和表情，但必须保留客户脸、年龄感、身体比例和真实身份。

使用建议：

- 普通客户图优先用 `动作保持`。
- 原片动作和新场景不匹配时用 `小幅调整`。
- 需要做完整风格大片、且能接受更高返工率时用 `重新设计`。

### 3. 填写任务表

使用 `workflow/batch_jobs.csv`。

规则：

- 你给多少张照片，就建多少行任务。
- 每行默认生成 1 张。
- 如果一张原片要出多个版本，修改 `variants`。
- 所有进入任务表的照片都按正式精修标准处理。

### 4. 主体锁定

正式生成前必须先确认保护区域：

- 脸部：禁止重绘，最终以原片脸为准。
- 头发主体：默认禁止被参考图发型污染；只有 `reference_hairstyle_option` 允许时才可参考。
- 身体姿态：优先保留原片。
- 动作表情：按 `pose_expression_option` 执行。
- 真实人物轮廓：优先保留原片。

### 5. 背景生成

背景可以按参考图生成，但必须先满足真实摄影标准：

- 真实光线
- 真实透视
- 真实景深
- 没有水印、文字、品牌标识
- 不出现参考图模特
- 不抢客户主体

### 6. 人物合成和光影融合

把原片人物放入新场景后，重点处理：

- 接触阴影
- 环境反光
- 边缘光
- 色温统一
- 皮肤与背景光线一致
- 婚纱和地面/花艺的真实遮挡关系

### 7. 原脸回贴和身份校准

如果任何生成步骤改变了脸，必须回贴原片脸部并重新融合光影。

通过标准：

- 像本人
- 不像参考图模特
- 不像 AI 重新捏出来的人
- 表情、脸型、五官、年龄感和原片一致

### 8. 商业精修

最终精修要求：

- 皮肤干净，但保留毛孔、肌理和微细节。
- 不塑料磨皮。
- 不假白。
- 不过度锐化。
- 婚纱边缘、发丝、手脚、背景穿帮必须处理。

### 9. 质检

使用 `qa/quality_checklist.md`。

任何一项触发红线，直接返工或废片。

## 批量生成规则

- 成本控制靠控制照片数量和 `variants`，不靠降低质量。
- 默认每张原片 `variants=1`。
- 重点照片可以设置 `variants=2` 或 `3`。
- 不批量放大风险。先验证一张做到真实，再扩展到整批。

## 强制禁止

- 禁止整图自由重绘客户。
- 禁止让参考图人物污染客户脸或发型。
- 禁止一眼假的背景贴图。
- 禁止人物浮在背景上。
- 禁止塑料皮肤。
- 禁止五官重绘、脸型改变、眼睛变大、鼻子变高、嘴唇变形。
- 禁止手指、腿部、婚纱边缘明显畸变。
- 禁止输出会损害门店客户信任的图片。
